# Advanced Media Player — WASM Firefox Extension

Real-time audio DSP Firefox extension with a Rust/WebAssembly core.

## Module Breakdown

### Rust WASM Core (`src/lib.rs`)

| Function | Description |
|----------|-------------|
| `apply_gain(samples, gain)` | Volume scaling with exponential soft-clamp at ±0.95 |
| `Equalizer` | Cascaded biquad filter chain (peaking, low-shelf, high-shelf) |
| `apply_compressor(samples, threshold, ratio, knee, attack, release, sample_rate)` | Dynamics compression with smooth knee and envelope follower |
| `time_stretch(samples, ratio, sample_rate)` | WSOLA-based algorithm for pitch-preserving speed change |
| `compute_spectrum(samples)` | Radix-2 FFT returning magnitude spectrum (half complex) |

### Extension Modules

**Content Script (`content.js`)**
- Captures `<video>`/`<audio>` elements via MutationObserver
- Creates `AudioContext` → `MediaElementSource` → `GainNode` → `AnalyserNode` chain
- Sends PCM frames to Web Worker for DSP processing
- Forwards `PROCESSED_AUDIO` and `MEDIA_STATE_UPDATE` to background
- Handles playback commands (`SET_PLAYBACK`, `SET_VOLUME`, `GET_MEDIA_STATE`)
- Site-specific next/prev selectors: YouTube, YouTube Music, SoundCloud, Spotify

**Web Worker (`worker.js`)**
- Loads WASM module dynamically
- Applies DSP pipeline: gain → EQ (cascaded biquad bands) → compressor → time-stretch → FFT
- Falls back to passthrough if WASM unavailable

**Background Service Worker (`background/`)**
- `main.js` — Central message router, tab lifecycle hooks, startup injection
- `state.js` — `managedAudioTabs[]` priority queue (index 0 = primary), `mediaStateCache`, `currentConfig`
- `tabs.js` — Enriches managed tabs with native browser metadata, lifecycle listeners for tab removal/navigation
- `media-session.js` — `MediaSession` API integration, config broadcast to all tabs, `pollActiveTab()` every 1.5s
- `injector.js` — `scripting.executeScript` wrapper for content script injection

**Popup UI (`popup/`)**
- `main.js` — Initializes all modules, listens for `MANAGED_TABS_MUTATED` and `SPECTRUM` events
- `player.js` — Primary tab transport controls, volume slider, seek bar with time display
- `tabs.js` — Renders background queue (index 1+), promote/focus/mute/close actions
- `equalizer.js` — 9-band peaking EQ sliders (60Hz–16kHz), compressor toggle, config dispatch
- `visualizer.js` — Canvas bar chart rendering FFT magnitude data with gradient bars
- `theme.js` — Persistent dark/light theme with system preference listener
- `store.js` — Module-level reactive state variables
- `ui.js` — DOM helpers, collapsible section persistence via `storage.local`

**Managed Tabs Queue System**

The background maintains a priority queue of media tabs:
- **Index [0]**: The primary tab driving the main player panel in the popup
- **Index [1+]**: Background queue displayed in the "Sound Tabs" section
- **Auto-promotion**: When the primary tab closes, the next in line automatically shifts to index 0
- **Manual promotion**: User can click any sub-tab to promote it to primary position

## Data Flow (Playback)

```
1. User visits YouTube → content.js captures <video>
2. Content creates AudioContext + GainNode + AnalyserNode
3. Content sends MEDIA_ELEMENT_CAPTURED → Background sets up MediaSession
4. Content sends MEDIA_STATE_UPDATE on timeupdate/play/pause events
5. Background stores state in managedAudioTabs[], broadcasts MANAGED_TABS_MUTATED
6. Popup receives signal → fetches GET_ALL_MANAGED_TABS → renders primary + queue
7. User adjusts EQ/volume/compressor in popup → sends UPDATE_CONFIG
8. Background broadcasts CONFIG_UPDATE to all tabs → content forwards to Worker
9. Worker applies DSP pipeline on PCM frames via WASM module
10. Worker posts SPECTRUM data → content forwards AUDIO_FRAME to background
11. Background broadcasts SPECTRUM to popup → canvas renders FFT bars
```

### Legend

| Entity | Role |
|--------|------|
| **Content Script** | Runs in page context. Owns AudioContext, captures media, bridges Worker |
| **Web Worker** | Dedicated thread. Loads WASM, executes DSP pipeline, posts spectrum |
| **Background** | Service worker. Central router, tab queue manager, config store, MediaSession |
| **Popup** | Transient UI. Player controls, EQ, visualizer, tab queue list |

## Build

```bash
# Prerequisites
rustup target add wasm32-unknown-unknown
cargo install wasm-bindgen-cli
cargo install wasm-opt  # optional, from binaryen

# Build
./build.sh

# Package
npx web-ext build -s extension/ --overwrite-dest
```
